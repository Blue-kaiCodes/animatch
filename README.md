# 🍥 AniMatch — Content-Based Anime Recommender

AniMatch recommends anime based on what you already like, using TF-IDF text
vectorization and cosine similarity over genres and synopses. Built with
Python, scikit-learn, and Streamlit.

**[Live demo](#)** *(add your Streamlit Community Cloud link here after deploying)*

## Why this project

Most "first repo" data science projects use the same handful of toy datasets
(Titanic, Iris). AniMatch instead works with real, messy MyAnimeList data —
deduplication, missing synopses, vote-count noise — and turns it into a
working, interactive tool rather than just a notebook.

## How it works

1. **Data cleaning** (`prepare_data.py`): collapses the raw one-row-per-genre
   dataset into one row per anime, filters out anime with too few votes to
   trust their score, and builds a combined text field weighting genres
   3x more heavily than synopsis text.
2. **Vectorization** (`recommender.py`): each anime's combined text is
   converted into a TF-IDF vector — rare, distinctive genres get more
   weight than common ones like "Comedy."
3. **Similarity**: cosine similarity between all anime pairs is precomputed
   once at startup.
4. **Filtering**: recommendations below a minimum MAL score are excluded, so
   you don't get "genre-similar but actually bad."
5. **App** (`app.py`): a Streamlit interface to search a title and view
   ranked recommendations with similarity scores.

## Running it locally

```bash
git clone https://github.com/<your-username>/animatch.git
cd animatch
pip install -r requirements.txt

# Regenerate the cleaned dataset (raw CSV not included in this repo —
# see "Getting the data" below)
python prepare_data.py

streamlit run app.py
```

## Getting the data

This project uses the open [tidytuesday anime dataset](https://github.com/rfordatascience/tidytuesday/tree/master/data/2019/2019-04-23),
itself sourced from MyAnimeList. Download it directly:

```bash
curl -O https://raw.githubusercontent.com/rfordatascience/tidytuesday/master/data/2019/2019-04-23/tidy_anime.csv
```

Place `tidy_anime.csv` in the project root, then run `python prepare_data.py`.

## What I'd improve next

- **Collaborative filtering**: incorporate user rating patterns (not just
  content) for a hybrid recommender — would likely outperform pure
  content-based matching for popular titles.
- **Better text features**: try sentence embeddings (e.g. `sentence-transformers`)
  instead of TF-IDF for the synopsis, which would catch thematic similarity
  TF-IDF misses (e.g. two synopses that are similar in meaning but use
  different words).
- **Cold-start handling**: TF-IDF works fine for existing anime but can't
  recommend brand-new releases with little synopsis text — would need a
  metadata-only fallback.
- **Scalability**: the current cosine similarity matrix is computed in full
  (fine at ~5,600 anime); a larger catalog would need approximate
  nearest-neighbor search (e.g. FAISS).
